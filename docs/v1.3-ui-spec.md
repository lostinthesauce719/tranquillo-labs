## 01 — Design System

All UI decisions derive from this single design language.

### Color Tokens

| Token | Value | Usage |
|-------|-------|-------|
| `--background` | `#F8FAFC` | Page background |
| `--surface` | `#FFFFFF` | Cards, panels, drawers |
| `--surface-mid` | `#F1F5F9` | Table rows alt, input backgrounds |
| `--border` | `#CBD5E1` | All dividers and outlines |
| `--text-primary` | `#0F172A` | Primary labels, headings |
| `--text-muted` | `#64748B` | Secondary labels, timestamps, placeholders |
| `--brand` | `#1B2A4A` | Nav, primary buttons, header backgrounds |
| `--accent` | `#C47B2B` | Urgency indicators, copper highlights |
| `--success` | `#166534` | Booked, confirmed, resolved states |
| `--warning` | `#92400E` | Pending, at-risk, follow-up states |
| `--danger` | `#991B1B` | Emergency, missed, escalated states |
| `--info` | `#1E40AF` | Informational badges, links |

### Typography Scale

- **Display / Page title:** 24px Semibold — dashboard header, empty state titles
- **Section heading:** 18px Semibold — card titles, drawer headers
- **Body:** 14px Regular — all descriptive content
- **Label:** 13px Medium — form labels, table headers
- **Caption / Mono:** 12px Regular — timestamps, IDs, phone numbers
- **Badge text:** 11px Semibold — all badge and chip labels

### Urgency Score Visual System

`urgencyScore` is a 1–10 integer. Must render consistently across every list, card, and detail view.

| Score | Label | Visual | Behavior |
|-------|-------|--------|-----------|
| 9–10 | EMERGENCY | Red filled badge + pulsing dot | Auto-escalation prompt, top of inbox, sound alert fires |
| 7–8 | URGENT | Amber filled badge | Sorted above standard, browser notification fires |
| 4–6 | STANDARD | Slate outline badge | Normal queue order, no alert override |
| 1–3 | LOW | Muted gray badge | Sorted below standard, informational only |

### Tier Gating — Universal Pattern

| State | What the user sees | CTA / Action available |
|-------|-------------------|----------------------|
| Feature visible, tier sufficient | Renders normally. No indicators. | Use feature |
| Feature visible, tier insufficient | Component renders with lock overlay + tier label. | Upgrade plan → opens /dashboard/billing |
| Add-on not activated | Component renders with "Not Active" overlay. | Activate add-on → opens /dashboard/settings#addons |
| Setup payment overdue | Full-screen gate on dashboard load. | Resolve payment → opens /dashboard/billing |

### Empty State Pattern

Every list, table, and queue must have a defined empty state. Template:
- Icon (lucide, 48px, muted color)
- Heading: what is empty ("No jobs booked yet")
- Body: why it might be empty + what to do
- CTA button when actionable

**Never show a blank table with column headers and zero rows.** Always render the empty state component.

## 02 — Shell Layout

The shell wraps every authenticated route. Three zones: Top Nav, Sidebar, and Content Area.

### Top Navigation Bar

- **Left:** Tranquillo wordmark + current location name (dropdown if multi-location, Enterprise only)
- **Center:** Global notification bell with unread count badge
- **Center:** Sound toggle (bell icon, persists to localStorage)
- **Right:** User avatar + role label chip + dropdown (Profile, Switch Role, Sign Out)

> Notification bell opens a slide-down panel: last 20 alerts, each linked to the relevant route. Mark all as read button.

### Sidebar Navigation

Fixed left sidebar, 240px wide, collapses to 56px icon-only on mobile.

| Nav Item | Icon | Roles | Badge / Indicator |
|----------|------|-------|-------------------|
| Dashboard | LayoutDashboard | All | None |
| Inbox | Inbox | Owner / Dispatch | Unread count — red if any EMERGENCY |
| Bookings | CalendarCheck | Owner / Dispatch | Count of today's upcoming bookings |
| Calls | Phone | Owner / Dispatch | Count of unreviewed calls |
| My Jobs | Wrench | Tech only | Count of today's assigned jobs |
| Settings | Settings | Owner only | Orange dot if setup incomplete |
| Billing | CreditCard | Owner only | Red dot if payment overdue |

> My Jobs only appears when logged-in user has role = tech. All other nav items are hidden for tech role except Dashboard.
> Settings and Billing only appear for owner role.

### Permission Hierarchy

Owner and Dispatcher access the same screens. Owner has override authority at every layer.

| Role | Screens | Can Edit | Cannot Do |
|------|---------|----------|-----------|
| Owner | All screens incl. Settings + Billing | Everything — override any dispatcher action | Nothing blocked |
| Dispatcher | Dashboard, Inbox, Bookings, Calls | Create/edit bookings, assign techs, update intake sessions | Settings, Billing, cannot change plan or team |
| CSR | Dashboard, Inbox | View and respond to intake sessions, create booking requests (not confirmed bookings) | Cannot confirm bookings, cannot assign techs |
| Tech | My Jobs only | Update own job status (En Route / On Site / Completed), add job notes | Cannot see other techs' jobs, cannot create or edit bookings |

### Real-Time Alert System

Two alert channels fire simultaneously when a new intake session arrives or urgencyScore ≥ 7.

1. **Live dashboard update** — Convex live query. Inbox badge, KPI counts, and activity feed update without page refresh. No polling.
2. **Browser notification** — fires if tab is not in focus. Requires one-time permission grant on first login. Shows customer name + issue summary + urgency level.

> Alerts fire for Owner and Dispatcher roles only. Techs receive a browser notification only when a job is assigned or reassigned to them specifically.
> Sound toggle in top nav controls an optional chime on browser notification. Off by default. Persists to localStorage per user.

## 03 — Onboarding Flow

A new company completing sign-up lands here before any operational screens. `setupStatus` drives gating.

**Route:** `/onboarding`

Full-screen stepped wizard. Not a modal. Sidebar is hidden. Progress bar at top shows 5 steps.

| Step | Title | Fields / Content | Completion triggers |
|------|-------|-----------------|---------------------|
| 1 | Company Profile | Company name, location name, timezone, phone number, business type (HVAC / Plumbing / Both / Other) | Write to companies table |
| 2 | Select Your Plan | Tier selector (Foundation / Ops Pro / Enterprise) with feature comparison. Monthly rate shown per tier. | Write companyPlan.tier |
| 3 | Setup Financing | If setup fee > 0: show payment options (Full pay / 50-50 / 3-pay / 6-month). Show total cost with premium. | Write financingPlan, setupBalanceRemaining |
| 4 | Team & Notifications | Invite team members (email + role). Grant browser notification permission (prompted here). Set after-hours start/end times. | Write memberships, notificationSettings |
| 5 | Activate Features | Toggle add-ons included in tier. Show locked add-ons with upgrade note. One-click activation for included features. | Write companyFeatures, set setupStatus = active |

- Back button available on steps 2–5. Step 1 cannot go back.
- User can exit after step 2 — account is created, plan is set, but features are inactive.
- Progress auto-saves at each step. Returning before completion resumes at last incomplete step.
- After step 5, redirect to /dashboard. Show welcome toast: "Tranquillo is active. Your first intake is ready."

## 04 — Screen Maps

### /dashboard — Owner / Dispatcher — All tiers

**Operations Overview**

Layout: Three-zone layout: KPI strip across top, main content grid below, right rail for activity feed.

**KPI Strip — 4 cards, full width**

| Component | Notes |
|-----------|-------|
| Calls Today | Total inbound calls. Subtext: X answered, Y missed. Links to /dashboard/calls |
| Open Intake Sessions | Count with urgency breakdown. Red if any EMERGENCY. Links to /dashboard/inbox |
| Bookings Today | Count of confirmed bookings for today. Links to /dashboard/bookings |
| Avg Response Time | Mean time from intake created to booking confirmed. Shown as minutes. |

**Main Content Grid — 2 columns**

| Component | Notes |
|-----------|-------|
| Upcoming Bookings | Next 5 bookings. Columns: Time, Customer, Service, Tech, Status badge. Row click → /dashboard/bookings |
| Recent Intake Sessions | Last 5 sessions. Columns: Channel badge, Issue summary, Urgency badge, Time. Row click → /dashboard/intake/[id] |

**Right Rail — Activity Feed**

Reverse-chronological event log for the current day
- Event types: intake received, booking created, call completed, escalation triggered, tech assigned
- Each entry: icon + short description + timestamp
- Max 50 entries. "View all" links to relevant route

**Tier-Specific Additions**
- Foundation: KPI strip only + activity feed. No table sections (feature locked with upgrade prompt)
- Ops Pro: Full layout as described
- Enterprise: Additional KPI card — Multi-location comparison (avg urgency by location)

**Empty States**

| State | What the user sees | CTA / Action available |
|-------|-------------------|----------------------|
| No data yet (day 1) | Welcome card with setup checklist: embed widget, add team, configure hours | Go to Settings |
| No bookings today | "No bookings scheduled for today" with calendar icon | Create manual booking |
| No intake sessions today | "No intake sessions yet" with inbox icon | View widget embed code |

### /dashboard/inbox — Owner / Dispatcher — Ops Pro+

**Intake Inbox**

Layout: Two-panel layout: session list on the left (360px), detail view on the right. On mobile, full-screen list → tap to open drawer.

**Left Panel — Session List**

Header: "Inbox" title + unread count badge + filter bar
- Filter options: All / New / Assigned / Emergency / By channel (Web / SMS / Voice)
- Sort: Urgency (default) / Newest / Oldest

**Session Card — anatomy**

| Component | Notes |
|-----------|-------|
| Urgency badge | EMERGENCY / URGENT / STANDARD / LOW — color-coded per design system |
| Channel badge | WEB / SMS / VOICE — icon + label. Slate outline style. |
| Customer name / phone | Name if captured. Phone number in mono caption below. |
| Issue summary | First 80 chars of issueRaw. Truncated with ellipsis. |
| Timestamp | Relative time (e.g. "4 min ago"). Absolute on hover. |
| Status indicator | Left border color: red = new/emergency, amber = in progress, green = resolved |
| Assigned tech chip | Shows tech initials if assigned. Empty if unassigned. |

**Right Panel — Session Detail**

Opens when a session card is clicked. Replaces right panel. Has its own close / back action.

| Component | Notes |
|-----------|-------|
| Session header | Channel, urgency score (large), status badge, time received, time to respond |
| Customer info | Name, phone, ZIP, propertyType, new vs returning indicator |
| Issue summary | issueRaw full text, aiSummary (if voice), serviceCategory badge, tags[] |
| Dispatch packet | accessNotes, pets, gateCode, dispatchNotes — structured, copy-friendly layout |
| Booking actions | Create Booking button (opens booking drawer). If booking exists: link to booking record. |
| Escalation banner | Only shown if urgencyScore ≥ 9. Red background. "This is an emergency — assign immediately." |
| Activity log | Status changes, notes, assignments for this session. Reverse chronological. |

**Actions Available from Detail View**

- Assign to tech — Select dropdown of active techs. Writes technicianName to booking.
- Create booking — Opens Booking Drawer
- Add note — Inline textarea, appends to activity log
- Mark resolved — Changes status, removes from active queue
- Escalate — Triggers immediate SMS to owner + moves to EMERGENCY priority

**Empty State**

No sessions: "Your intake queue is clear." with inbox icon and instruction to test the embed widget.

### /dashboard/settings — Owner only — All tiers

**Settings**

Tab-based layout. Six tabs. Changes save immediately (no save button) except for Intake Configuration which has an explicit save.

**Tab 1 — Company**

- Company name, location name, timezone, business phone
- Business type checkboxes: HVAC / Plumbing / Electrical / Roofing / Other
- Operating hours — per day of week, open/closed toggle + time range picker
- After-hours mode start/end times (requires afterHoursMode add-on)

**Tab 2 — Team**

| Component | Notes |
|-----------|-------|
| Member list | Name, email, role badge, status (Active/Invited/Inactive), actions |
| Invite member | Email input + role select (dispatcher / csr / tech). Sends Clerk invitation. |
| Edit role | Inline role change. Cannot demote the only owner. |
| Remove member | Confirm Dialog before removal. Reassigns their open bookings first. |

**Tab 3 — Notifications**

- Browser notification permission status — shows "Granted" / "Blocked" with instructions to unblock if needed
- Notification test button — fires a test browser notification immediately
- Sound alerts: toggle on/off (sets default for all users, each user can override in their nav)
- Emergency contact phone — shown to techs on emergency jobs. Not used for alerts.

> SMS and email alert channels are not part of v1.3. Notification delivery is browser-only.

**Tab 4 — Intake Configuration**

Only visible for Ops Pro and above. Shows locked state for Foundation.

- Service categories active for this company — checklist
- Emergency keywords — editable tag list with defaults shown
- Custom intake questions — add/remove/reorder per service category

Each add-on shown as a card: name, description, status, monthly cost, activate/deactivate button.

- Active add-ons: green badge, Deactivate button (with confirm: data is retained)

- Locked by tier: lock icon, "Included in [tier] plan" or "Upgrade to unlock"

**Tab 6 — Branding**

- Company display name (shown to customers in SMS/intake widget)
- Primary color hex input (widget accent color)
- Logo upload (shown in widget header)
- Widget preview — live iframe of the embed widget with current settings

| Meter | Notes |
|-------|-------|

## 05 — Web Intake: Widget + Hosted Page

Web intake ships in two formats from the same underlying component. Both are active for any company with the webIntake add-on enabled.

| Format | Description |
|--------|-------------|
| Embeddable Widget | Script tag injected into client's own website. Renders floating button + iframe overlay. |
| Hosted Browser Page | Standalone page hosted at /intake/[companySlug]. Works as a direct URL. |

**Use cases:**
- Widget: Client pastes into their existing site — Wix, Squarespace, WordPress, GoDaddy builder.
- Hosted: Share as a link in Google Business, SMS campaigns, QR codes, social profiles.

**Branding:** Both use client logo + primary color from Settings > Branding.

**Embed Code (Widget)**

One script tag with a data-company-id attribute. Injects a floating button + iframe. Company branding pulled from Settings > Branding.

> Embed code available in Settings > Intake Configuration > Widget Embed Code. Copy button. Includes installation instructions for common platforms.

**Hosted Page Route**

Route: `/intake/[companySlug]` — public, no authentication, server-side rendered for SEO and fast load.

- companySlug set during onboarding. Editable in Settings > Company (with redirect from old slug).
- Page title: "[Company Name] — Request Service"
- Meta description uses company name + service types for local SEO benefit
- Share URL shown in Settings > Intake Configuration alongside embed code

**Widget + Hosted Page — Shared States**

| State | What the user sees | CTA / Action available |
|-------|-------------------|----------------------|
| Closed | Floating button (bottom-right). Shows company logo or "Get Help" label. | Click to open |
| Open — Greeting | Intake form slides up. Company name + "How can we help?" header. | Select service type |
| Open — Intake flow | Step-by-step question form. Progress bar at top. Adaptive per service. | Answer questions |
| Open — Address step | Google Places autocomplete. City/state/zip locked after selection. | Confirm address |
| Open — Confirmation | Summary of captured info. Time slot selection. | Confirm booking |
| Submitted | Success state: "Your appointment is confirmed." + SMS confirmation note. | Close widget |
| Emergency detected | Red banner: "This sounds urgent. Calling you now." + phone number shown. | Call now button |
| Outside service area | Soft warning: "We may not serve your area. We'll confirm." | Continue anyway |
| After hours | After-hours message. Still captures intake. Flags for morning review. | Leave your info |

**Intake Flow — Step by Step**

Rules-first branching. LLM only used for aiSummary generation after data is collected.

| Step | UI | Logic / Branch |
|------|-----|----------------|
| 1 | Service type | Button grid: HVAC / Plumbing / Electrical / Other |
| 2 | Issue description | Textarea + common issue chips (e.g. "No heat", "Clogged drain") |
| 3 | Emergency check | If emergency detected: "Is this an active emergency?" Yes/No |
| 4 | Property type | Button grid: House / Apartment / Condo / Commercial |
| 5 | Address | Google Places autocomplete. Unit field always editable. |
| 6 | Access details | Checkboxes: Gate code / Pets / Parking notes + text fields |
| 7 | Contact info | First name, phone number (required), email (optional) |
| 8 | Time preference | Select preferred window. Shows "We'll confirm by SMS." |
| 9 | Review + submit | Summary card of all captured info. Edit links per section. |

> Creates intakeSession. Sends SMS. Notifies dispatcher.

**Widget Mobile Behavior**

- Full-screen overlay on mobile (not a floating panel)
- Back button on every step
- Keyboard-aware — form scrolls above keyboard on iOS/Android
- Step progress shown as dots at top (not percentage)
- Google Places autocomplete tested on mobile Safari and Chrome
